Detailed Description

The RHI fence implementation for Metal. This exists separately from Fence to decouple the RHI::Device instance from low-level queue management. This is because RHI::DeviceFence holds a reference to the RHI device, which would create circular dependency issues if the device indirectly held a reference to one. Therefore, this implementation is only used when passing fences back and forth between the user and the RHI interface. Low-level systems will use the internal Fence instance instead.
